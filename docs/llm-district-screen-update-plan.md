# LIBscope 자치구별 현황 LLM 하네스 화면 반영 계획

기준일: 2026-06-09

## 목적

자치구별 현황 화면의 모든 변동성 지표를 LLM 연결 전 단계에서 동일한 판단 단위로 분리하고, 이후 모델 키가 추가되면 mock 결과를 실제 생성 결과로 교체할 수 있는 구조 구축.

## 화면 반영 범위

1. 자치구 종합 인사이트
   - 위치: 자치구 화면 최상단
   - 입력: 인구구조, 문화역량, 교육인프라, 사회안전망 해석 결과
   - 출력: 3개 카드형 문장
   - 현재 상태: `/api/llm-harness` mock 응답 사용

2. 자치구 보고서 미리보기
   - 위치: 자치구 선택 박스 우측의 `자치구 보고서 미리보기` 버튼
   - 입력: 지표묶음별 해석 결과 전체
   - 출력: 보고서 섹션 배열, HTML 문자열, Markdown 문자열
   - 현재 상태: 화면 내 미리보기 및 Markdown 다운로드 지원

3. 인구구조 분석
   - 위치: 연령대별 인구 분포/성별 비율 차트 하단
   - 입력: 주민등록 인구, 생활인구, 연령대, 성별 구성
   - 출력: 짧은 해석문, 핵심 확인값, 유의사항
   - 문체: 보고서형 명사형 어미

4. 문화역량·향유 지표
   - 위치: 문화 역량·향유 지표 섹션 하단
   - 입력: 2023 문화지표 자치구 값, 2024 문화향유 집단별 참고값
   - 출력: 고정 데이터셋 사전 생성형 해석문
   - 운영 방향: 최초 생성 후 DB 저장, 보고서 생성 시 지표값과 함께 재사용

5. 교육인프라
   - 위치: 학교급별 목록 하단
   - 입력: 학교급별 개수, 학교명/주소 목록, 공공도서관 수, 문화행사 수
   - 출력: 교육기관 규모와 도서관 협력 가능성 해석

6. 사회안전망 대상자 구성
   - 위치: 가구/장애/외국인 구성 박스 하단
   - 입력: 수급률, 가구 구성, 장애 대분류, 외국인 주민 유형, 등록외국인 국적
   - 출력: 접근성 점검 관점의 해석문
   - 기준: 대상자 특성을 단정하지 않고 이동, 언어, 정보 도달성, 이용 장벽 관점으로 표현

## 현재 구현 파일

- `functions/_shared/llm-harness.cjs`
  - mock 해석 생성 엔진
  - 문체/출력 가이드
  - snapshot key 생성

- `functions/_shared/llm-model-registry.cjs`
  - provider 카탈로그
  - 모델별 비용 등급과 역할
  - 섹션별 기본/승격 모델 추천값

- `functions/_shared/llm-output-contracts.cjs`
  - 섹션별 입력·출력 계약
  - 문체·근거 규칙
  - 품질 게이트
  - 보고서 목차와 골든 테스트 자치구

- `functions/llm-harness.js`
  - `/api/llm-harness`
  - GET: 모델 registry, 출력 계약, 모델 추천 및 키 준비 상태
  - POST: 자치구 화면용 mock 해석/보고서 생성

- `src/App.jsx`
  - 자치구 종합 인사이트 mock 연결
  - 보고서 미리보기 및 Markdown 다운로드
  - 지표묶음별 해석 패널 추가
